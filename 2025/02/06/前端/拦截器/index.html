拦截器

一、什么是拦截器

在前端中，拦截器主要用于在HTTP请求和响应过程中进行统一处理。也就是，在发起请求和接受响应时，对数据处理。形象点说，先将数据和操作拦截下来，进行一定地处理。举个例子，你要去吃饭，但你妈把你拦住，要求你先洗手再去吃饭，这就相当于一个请求拦截器；你去吃饭，吃完饭后你妈妈拦住你，要求你把碗筷收拾好、擦干净桌子，然后才能离开餐桌。这个过程就相当于响应拦截器。

响应拦截器

  1. 统一数据格式处理 假设后端返回 { code: 200, data: {}, message: "success" }，拦截器自动剥离外层结构，直接返回 data 字段。若业务逻辑错误（如 code=400），则抛出异常。
  2. HTTP 错误状态码处理
    • 401：跳转登录页
    • 403/404/500：提示用户友好信息
    • 网络错误：提示网络问题
  3. 自动重试机制 对失败请求自动重试（如网络波动场景），最多重试 2 次，延迟 1 秒。
  4. 响应日志记录 记录成功和失败的响应信息，便于调试和监控。

2、请求拦截器

  1. 添加公共请求头 自动为所有请求添加 Authorization 头（从 localStorage 读取 Token），适用于身份验证。
  2. 取消重复请求 当同一请求（相同 URL + 方法）重复发送时，自动取消之前的请求，避免资源浪费。
  3. 防止 GET 请求缓存 为 GET 请求添加时间戳参数 _t，避免浏览器或服务器缓存旧数据。
  4. 请求日志记录 在控制台输出请求的详细信息，便于调试和监控。

三、拦截器功能和应用场景

1、请求拦截器

  • 添加通用请求头：如认证令牌（Authorization）。
  • 请求参数处理：如序列化数据、添加时间戳。
  • 请求日志：记录请求信息，便于调试。
  • 请求取消：取消重复或无效请求。

2、响应拦截器

  • 统一错误处理：如处理HTTP错误状态码。
  • 数据格式化：统一处理响应数据格式。
  • 响应日志：记录响应信息，便于调试。
  • 重试机制：在请求失败时自动重试。

3、应用场景

  • 身份验证：自动添加或刷新令牌。
  • 加载状态管理：显示或隐藏加载动画。
  • 错误提示：统一处理并提示错误信息。
  • 数据缓存：缓存响应数据，减少重复请求。
